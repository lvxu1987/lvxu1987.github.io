启动V2Ray时提示端口被占用

By 吕旭

发表于 2020-04-19

昨天启动我电脑上的V2RayN软件时，无法正常启动，弹出以下错误提示。

Privoxy Error
✖Fatal Error: can’t bind to 0.0.0.0:1081:(error number 0)

查了一下，是端口被占用的原因。

软件端口冲突问题

网上查询了一下，有可能是软件端口冲突，需要看看是哪个软件占用了该端口。打开CMD命令行，在里面输入：netstat -ano | findstr "1081"，但是系统没有任何输出。难道是我的命令输错了么？尝试另一个端口，发现是有输出的。

而且使用tasklist | findstr "13008"命令，还能看到占用了13008端口的软件是我的BT下载软件Transmission。

至此说明问题并非软件端口冲突导致的。

又在网上搜索，发现有人说是因为TCP/IP协议或Winsock损坏。

TCP/IP协议或Winsock损坏问题

按照网上的说法，以管理员身份启动命令提示符，然后执行：

三条命令执行完毕后，重新启动计算机。发现故障依旧。

突然想到我之前安装了Hyper-V，会不会端口被系统保留了？

系统保留端口

打开命令提示符，输入netsh interface ipv4 show excludedportrange protocol=tcp，发现1080和1081端口果然被保留了。

尝试退出关闭Hyper-V管理器后，输入netsh int ipv4 add excludedportrange protocol=tcp startport=<起始端口> numberofports=<端口保留数量>这个命令来让用户保留相关的端口。

用管理员权限打开命令提示符，尝试之后却发现系统提示错误。

原来简单的关闭Hyper-V管理器并不能让Hyper-V退出运行，必须禁用这个功能才行。

在命令提示符里输入dism.exe /Online /Disable-Feature:Microsoft-Hyper-V,系统出现如下提示：

重新启动后，再执行netsh interface ipv4 show excludedportrange protocol=tcp命令，发现系统已经不再保留1081端口了。

再次执行netsh int ipv4 add excludedportrange protocol=tcp startport=1080 numberofports=4命令后，系统已将1080-1083这4个端口保留为用户使用。

尝试启动V2RayN软件，故障已排除。

接下来，就是执行dism.exe /Online /Enable-Feature:Microsoft-Hyper-V /All命令并重启计算机，再次启用Hyper-V功能。

重新启用Hyper-V以后电脑无法联网

重启之后，发现电脑竟然无法联网了。打开更改适配器设置的界面，发现原先里面的Hyper-V的vEthernet虚拟交换机不见了。

我之前安装Hyper-V以后，配置了一个外部虚拟交换机，该虚拟交换机通过物理网卡获得IP地址，真机和虚拟机相当于都连接在了这个虚拟交换机上，虚拟机也可以通过这个虚拟交换机获得外部网络的IP地址。

现在这个虚拟交换机不见了，真机自然也就断网了。

找到问题的原因，解决办法就很明确了，重新建立一个虚拟交换机就好了。不过事情就是这样，总是不会顺利，在Hyper-V里面竟然无法重新建立新的外部虚拟交换机，Hyper-V提示物理网卡已经被占用了。可是我的虚拟交换机已经不见了啊？

查看物理网卡的属性，发现Hyper-V可扩展的虚拟交换机为被选中的状态，将其取消选中就可以取消占用了。

再次尝试新建外部虚拟交换机，已经可以重建了。

至此，真机、虚拟机都可以正常上网，V2RayN也可以正常使用。故障完全排除。
